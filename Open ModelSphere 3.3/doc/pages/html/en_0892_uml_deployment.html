Deployment Diagram





The Deployment diagram describes the configuration of processing resource elements and how software implementation components map onto them.

The following concepts can be used to build Deployment diagrams with Open ModelSphere:
  • Component
  • Node
  • Dependency

To add a use case

  • Select the Component creation tool located in the UML modeling toolbar.
  • With the Component tool selected, click anywhere inside the diagram window.


To add a Node 
  • Select the Node creation tool located in the UML modeling toolbar.
  • With the Node creation tool selected, click anywhere inside the diagram window.

To add a Dependency
  • Select the Dependency creation tool located in the UML modeling toolbar ( or )
  • Choose one of the Dependency displayed in the toolbar.
  • With the Dependency tool selected, click somewhere in the middle of the source Component, then, in the middle of the target Component.

Assign a UML sterotype to an object
  • Select any object from the diagram or from the explorer.
  • In the design pannel, choose from the list UML Sterotype.



Assign a UML constraint to an object
  • From the explorer or from the diagram right-click an object and click the properties tool
  • Select the UML Constraint tab, click the Link button.
Select a UML Constraint from the pop-up window.

The common concepts such as Semantic Links and Graphical Notes can also be used to enrich your Deployment diagrams.
