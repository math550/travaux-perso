Grouper


La commande Grouper facilite le travail d’équipe. Elle offre la possibilité de créer différents modèles en parallèle pour les regrouper par la suite.

Pour exécuter un regroupement, le modèle cible et le modèle source doivent être ouverts. On doit d'abord sélectionner le processus dans le modèle source, celui que l'on veut greffer au modèle cible. Ce processus et tout ce qui se trouve en dessous sera groupé à la place d'un processus externe dans le modèle cible.

On lance l'opération en choisissant dans le menu principl: Outils > Grouper. Cette commande fait apparaître la fenêtre Options de groupement.

Cette fenêtre offre les options suivantes :
  1. Sous : cette boîte de saisi permet de choisir le processus qui recevra la branche à greffer.

  2. Mettre à jour à partir de la source : quand cette option est cochée, les concepts du modèle cible qui ont le même identifiant (identifiant sémantique) que ceux des concepts du modèle source héritent de toute l’information des concepts du modèle source.

  3. Grouper les qualificatifs : la liste des qualificatifs pour les concepts ayant les mêmes identifiants sera la somme des qualificatifs dans chaque concept avant le regroupement.

  4. Grouper les ressources : la liste des ressources pour les concepts ayant les mêmes identifiants sera la somme des ressources dans chaque concept avant le regroupement.

  5. Concaténer les descriptions différentes : identique à l’option Grouper les qualificatifs, mais agit sur les descriptions. Si le champ de la description diffère de quelque façon, la description du modèle source sera ajoutée à la fin de celle du modèle cible.

Cliquez sur le bouton OK pour déclencher le processus de regroupement. Quand l’opération est complétée, vous devriez obtenir le message suivant : L’opération s’est terminée avec succès. Si tel n’est pas le cas, lisez le fichier d’erreurs, apportez les corrections nécessaires au modèle et recommencez l’opération. Après l’opération, le processus regroupé devient un sous-processus du modèle récepteur.


Reconnaissance des concepts communs

Quand le processus à greffer comporte des liens avec d'autres processus de même niveau dans le modèle de destination, il est important de s'assurer des points suivants afin qu'il fasse un lien et d'éviter qu'il duplique le même processus.
  • Les chaînes hiérarchiques des identifiants numériques ou alphanumériques doivent être complètes.

  • On peut utiliser soit les identifiants numériques ou alphanumériques, dans la mesure que ce soit la même chose tant pour la source que pour la cible.

  • Les identifants pour le niveau de greffe doivent être les mêmes des deux côtés pour pouvoir être reconnus.
    (ex. Si on greffe un processus 2 qui est relié à un processus 3, sur un processus 4.1.2, un lien sera effectué si un processus 4.1.3 existe dans le modèle cible. Autrement il sera créer.)

Il en va de même pour les entités externes, les dépots et les flux.
