Changing Target System


If you want to change the DBMS on which a database model is implemented, ModelSphere supports to migrate a model to an other target system, assuring the data type correspondence.

To invoke the changing target system feature, right-click a data model, a database, a user-type model or an operation library in explorer and select the 'Changing Target System...' item in the pop-up menu.

A wizard dialog then appears.



The first page of the wizard allows to select the destination target system. Hit the browse button to select it. Click Next to continue.





At this step, you have to determine if you want to migrate only this module (here a data model, but that can be also a database or an operation library) or all the database deployment. Click Next to continue.





You have to decide if you want to convert the submodels and if you want to keep the actual linked common items. Click Next to continue.





A report of what is about to be performed is displayed and right after the conversion process will be done. Click Next to process.
