Converting a relational data model to a conceptual data model

A relational data model can be converted to a conceptual model. Tables are turned into entities, connectors into relationships etc. on a 1:1-basis. Existing relational descriptors that do not apply to conceptual models are not reflected in the resulting diagram and are greyed out in the properties windows.

It is possible to convert a relational data model from the main menu, from the explorer window and from the diagram.

  • From the main menu, choose Tools > Convert to a Conceptual Model....
  • From the explorer window, right-click the model (or the model's diagram). Then, from the pop-up menu, choose Convert to a Conceptual Model....
  • From the diagram, right-click in the diagram,choose Convert to a Conceptual Model....


The conversion process creates a new model. You can choose

  • To only convert the model in the active diagram or include all its submodels as well
  • The notation for the target model

Please note to

  • Cleanup the model
  • Verify integrity
before the conversion.

A relational model and its converted model are independent from each other, i.e. modifications made after the conversion will not be reflected in the converted model. Each following conversion will not update a previous result, but create another conceptual model.
