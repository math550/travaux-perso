Modélisation de données et de classes Java


Les concepteurs de systèmes d'information analysent les organismes depuis longtemps et ils continueront dans l'avenir. Ainsi, pendant plusieurs années, le modèle relationnel était le paradigme dominant pour la communication (modèle relationnel de données) et l'implémentaton (systèmes de gestion de base relationnelle de données). Suivant l'apparition du langage de modélisation unifiée (UML) et de la maturation des technologies orientées-objet, les concepteurs ont commencé à utiliser l'analyse orientée-objet (OO) et à créer des modèles pour ces nouveaux types de systèmes d'information. La situation actuelle (les modèles relationnels et d’objets étant tous deux utilisés) doit être gérée d’une manière interactive afin d’être profitable.

Une des caractéristiques principales de Open ModelSphere est la combinaison de la gestion de modèles, de modélisation de données relationnelles standard et de modélisation de classes basée sur UML. L'objectif est de fournir le mécanisme de relations et les fonctionnalités les plus flexibles pour créer des modèles.
