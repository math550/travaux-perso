Glisser-Déposer


Afin de rendre le processus de création de diagrammes plus efficient, ModelSphere vous permet de glisser-déposer des objets sur un diagramme. La section ci-dessous décrit comment utiliser cette fonctionnalité.

Déplacement d'objets avec glisser-déposer

Dans la pluspart des cas, la fonctionnalité glisser-déposer va déplacer un objet (objet source) à partir d'un conteneur dans un nouveau conteneur (conteneur cible). Les seules exceptions sont l'assignation d'une valeur à un objet et la création d'une représentation graphique. En utilisant la fonctionnalité glisser-déposer, l'objet source provient toujours de l'explorateur. Cependant, le conteneur cible peut être l'Explorateur ou la fenêtre de diagramme.
  1. De la fenêtre explorateur, sélectionnez un objet source avec la souris.


  2. Tenir le bouton de la souris et glissez la ressource sur un processus se trouvant dans le diagramme ou l'explorateur.


  3. Déposez l'objet source en relâchant le bouton de la souris quand le conteneur cible devient sélectionné et que l'icone sous le curseur de la souris devient '+'.
Glisser-Déposer assignation d'une valeur à un objet

Il est possible d'assigner des ressources, qualifiquatifs et items communs à différents objects en utilisant la fonctionnalité glisser-déposer. Pour ces cas spéciaux, la fonctionnalité glisser-déposer assignera l'objet source (ressource, qualifiquatif ou item commun) à un conteneur cible plutôt que de déplacer l'objet source.

Glisser-Déposer création d'une representation graphique

En utilisant la fonctionnalité glisser-déposer à partir de l'explorateur vers le diagramme créera une représentation graphique de l'objet dans le diagramme. Ceci ne se produira que si le conteneur cible est le diagramme et aussi si l'objet source peut être représenté graphiquement.

Règles pour la fonctionnalité glisser-déposer

Dans le but de garder l'intégrité des modèles, quelques règles s'appliques.
  1. Si l'icone sous le curseur de la souris ne devient jamais '+', ceci signifie que le conteneur cible n'est pas valide pour l'objet source.


  2. La fonctionnalité glisser-déposer ne peut être utilisée à travers 2 projets.


  3. Un objet source peut être convertit pour dépendament de sont conteneur cible. Par exemple, une table oracle perdra ses paramètres de stockage si elle est déplacée vers un modèle de données SQL Server.


  4. Pour que le conteneur cible soit valide, il faut qu'il soit de même type que sont conteneur source. Par exemple, il est possible de glisser-déposer une colonne vers une table. Cependant, il n'est pas possible de glisser-déposer une colonne vers un modèle de classe.
