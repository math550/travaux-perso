Les formalismes
 

Les formalismes sont l’ensemble d'options d'affichage graphique utilisés pour associer les éléments du modèle. Il y a plusieurs formalismes prédéfinis. Cependant, vous pouvez personnaliser des formalismes existants ou créer les vôtres. Si vous préférez ne pas les écraser, votre projet aura les formalismes par défaut qui s'appliquent à tous les diagrammes.
 

L'héritage

Les diagrammes héritent d’un formalisme par défaut, à moins que l'utilisateur ne remplace leur valeur. Si la valeur d’un formalisme est changée dans le projet, elle changera aussi dans le diagramme.
 

Introduction des formalismes prédéfinis

Open ModelSphere vous permet de travailler avec quatre formalismes prédéfinis pour les modèles de données ( Datarun, Information Engineering, Logical Data Structure et UML) ainsi qu’un bon nombre pour les modèles de processus (Gane Sarson, Yourdon-DeMarco, Merise, etc.).
Ces formalismes prédéfinis ne peuvent pas être modifiés ou personnalisés. Par contre, vous pouvez créer vos propres formalismes. Pour ce faire, voir la section qui suit.

Vous pouvez changer de formalismes à tout moment pendant votre session de travail. Pour changer de formalismes :

     
  1. Assurez vous que le projet est en sélection active (mis en évidence)
  2. Du menu principal choisissez Format > Formalismes par défaut. Une liste apparaîtra.
  3. Choisissez le formalisme à utiliser en cliquant dessus.
Les formalismes définis par l'utilisateur

Formalismes pour la modélisation de données

Il est possible de personnaliser des formalismes existants ou de définir les vôtres. Vous pouvez créer un formalisme à partir d'un autre ou en créer un nouveau. Choisissez Format > Formalisme du projet. Vous obtiendrez ainsi la fenêtre des formalismes, dans lequel vous pourrez personnaliser les formalismes.

Il est possible de supprimer, d'ajouter de nouveaux formalismes ou d'en créer à partir de formalismes existants. Les options d’affichage et de position peuvent être modifiées ici. Quand tout est réglé, cliquez sur Appliquer afin de valider votre sélection, puis Fermer.
 

Formalismes pour la modélisation de processus

Open ModelSphere offre plusieurs formes prédéfinies pour les processus, entités externes et dépôts.

Modifier la forme des Processus, Entités externes et Dépôts

Les formes de vos objets graphiques seront différents selon le formalisme que vous aurez choisi.

Contrairement aux dépôts et aux entités externes où le préfixe est seulement ajouté graphiquement à l'identifiant, le préfixe du flux est utilisé dans la création de son identifiant. Les changements du préfixe du flux seront effectifs à la création suivante d'un flux.
