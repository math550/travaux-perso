La base de données


Le concept de base de données a une double utilisation. En soi, c’est un conteneur pour les concepts physiques (comme les tablespaces), mais il agit aussi comme un noeud de déploiement en reliant les modèles de données, les modèles de domaines et les bibliothèques d'opérations qui ont un système cible commun.

Pour ajouter une base de données, sélectionnez projet > Ajouter > Ajouter une base de données > système cible de votre choix.


Les modèles déployés




Les options obligatoires

Le modèle de données
  • Vous pouvez associer un seul modèle de données à une base de données.
  • Cette association est obligatoire lorsque vous utilisez une interface SGBD. Certains éléments du modèle de données peuvent être reliés aux objets physiques de la base de données (table, tablespace). L’application utilise l’association modèle de données - base de données afin de déterminer les objets physiques disponible pour le lien.
Les modèles de domaines
  • On peut lier deux modèles de domaines selon deux approches: le modèle type utilisateur et le modèle de domaines par défaut
  • Ces associations sont facultatives.
  • Le modèle type utilisateur est le contenant des types de données définis par l'utilisateur. Cette association est obligatoire si ces types sont générés par une interface SGBD.
  • Le modèle de domaines par défaut est le modèle de domaines qu'utiliseront certaines fonctionnalités qui auront à générer des domaines.
La bibliothèque d’opérations
  • On peut lier une seule bilbiothèque d'opérations à une base de données.
  • Cette association est obligatoire pour être utilisé par une interface SGBD.
Ces liens sont utilisés dans des fonctions (validation et génération) et dans l’association d'éléments de modèles avec les objets physiques de la base de données.

Afin de définir les tablespaces, le modèle de données doit être relié à la base de données qui contient l’information quant à ces tablespaces.
