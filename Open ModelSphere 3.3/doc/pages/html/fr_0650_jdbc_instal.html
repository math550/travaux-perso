Installation d'un pilote JDBC


Pour rendre Open ModelSphere capable de se connecter à une base de données, un pilote JDBC est requis. Il y a deux sortes de pilote JDBC : Les pilotes JDBC natifs, spécifique à un SGBD et le pilote générique JDBC-ODBC.


Les pilotes JDBC natifs

Les pilotes natifs peuvent être obtenus sur le site web du fournisseur de SGBD, ou disponible lors de l'achat de ceux-ci. Contactez votre fournisseur pour plus de détails. Vous en trouverez ici quelques exemples.

Système cible Fichiers de pilote Classe du pilote Où se les procurer
DB2 db2java72.zip COM.ibm.db2.jdbc.net.DB2Driver Vient avec le SGBD
Informix ifxjdbc.jar com.informix.jdbc.IfxDriver Vient avec le SGBD
MySQL mysql-connector-java-3.0.8-stable-bin.jar com.mysql.jdbc.Driver http://www.mysql.com/products/connector-j
SQL Server mssqlserver.jar | msbase.jar | msutil.jar com.microsoft.jdbc.sqlserver.SQLServerDriver Vient avec le SGBD
Oracle jdbc901-thin-classes12.zip oracle.jdbc.driver.OracleDriver http://www.oracle.com/technology/tech/java


Quand vous avez obtenu le fichier pilote (habituellement un fichier .jar ou .zip), copiez-le dans le sous-répertoire: / jre / lib / ext de votre installation Java. Redémarrez ModelSphere.

Chaque pilote JDBC a son propre nom de classe. Consultez la documentation relative à votre SGBD pour le connaître. Ceci sera essentiel pour utiliser le pilote JDBC. Voir Connexion à une base de données avec JDBC.


Le pilote générique JDBC-ODBC

Le pilote JDBC-ODBC s'installe avec l'application ModelSphere. Ce pilote peut être utilisé avec une source de donnée ODBC déjà définie. Le type de connexion offert par ce pilote est uniquement compatible avec le système cible générique.
