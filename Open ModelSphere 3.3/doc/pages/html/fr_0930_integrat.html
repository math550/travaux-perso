La comparaison et l'intégration de modèles
 

L'intégration est le processus interactif dans lequel deux modèles sont comparés et peuvent ensuite être synchronisés.

Cette fonction procède avec l'information sémantique et les modèles du même type. Voici tous les types modèles qui peuvent être intégrés.
 

Types de modèle

     
  • Modèle d'items communs
  • Modèle conceptuel de données
  • Modèle relationnel de données
  • Modèle de domaines
  • Bibliothèque d'opérations
  • Base de données
  • Modèle de processus
  • Modèle de classes
  • Paquetage (class modeling)


Pour intégrer des modèles :

     
  1. Choisissez le modèle à intégrer.
  2. Selectionnez Outils > Comparer/Intégrer... dans le menu principal.
  3. Une fenêtre va apparaître.

Spécifications : Avec un modèle de données, l’intégration créera des liens hiérarchiques si vous comparez des sous et super-modèles.
 

Les options

Utiliser comme identifiant : le processus d'intégration emploie les noms d'objet ou les noms physiques dans le processus de comparaison et l'identifiant est affiché dans la fenêtre d'intégration.

Portée : le processus applique un filtre aux concepts et aux descripteurs qui sont comparés. Cette liste est basée sur un ensemble de concepts qui sont compatibles en ce qui concerne le type et le système cible des modèles choisis. Il est possible de sauvegarder et recharger les paramètres de portée d'intégration (.scp).

Comparer les objets externes en utilisant (avancé) : un objet externe est celui qui n'est pas inclus dans les deux modèles choisis pour l'intégration. Par exemple, en intégrant deux modèles de données, il est probable que les colonnes feront référence aux domaines qui ne font pas partie d'un modèle de domaines spécifique. Le domaine est un objet externe.

  • [Identifiant] : l'intégration fait rapport d’une différence si les deux références externes indiquent deux objets différents n’ayant pas le même nom ou nom physique. Dans notre exemple avec le modèle de données, si le type de données des deux colonnes est lié à des domaines différents ayant le même nom, aucune différence ne sera rapportée pour le type. Bien sûr, si les domaines ont des noms différents, on détectera une différence.

    •  
  • [Référence] : l'intégration fait rapport d’une différence si la référence à l’objet externe n'indique pas exactement le même objet (comparé par référence). Dans notre exemple avec le modèle de données, si le type de données des deux colonnes est lié à des domaines différents, l’application détectera une différence automatiquement (peu importe si le nom ou nom physique sont les mêmes).

    •  


Le fonctionnement de l’opération

Le processus d'intégration utilise des liens hiérarchiques pour associer les objets et leur nom ou leur nom physique, selon la valeur choisie.

La partie supérieure de la fenêtre d'interface affiche l'information concernant les modèles comparés. Vous remarquerez que la vue ressemble à celle de l'Explorateur.

BLEU : l’objet n’est que dans le modèle 1
VERT : l’objet n’est que dans le modèle 2
ROUGE : objets différents
NOIR : objets identiques

Le partie inférieure de la fenêtre affiche les différences, s’il y en a, entre les propriétés des objets.
 

Les actions applicables aux objets

Après avoir comparé les deux modèles et avoir affiché leurs différences, l'utilisateur peut modifier les modèles de façon sélective et ceci dans n'importe quelle direction. Cependant, il est important de spécifier que toutes les actions sont appliquées hiérarchiquement sur le choix d'objet. Voici les actions offertes :

     
  • Fusionner : les objets du modèle source sont ajoutés au modèle cible afin de former un super-ensemble de toutes les composantes du modèle.

    •  
  • Remplacer : l'information du modèle cible est remplacée par celle du modèle source. Toute information supplémentaire dans le modèle cible est supprimée.

    •  
  • Aucune : aucune action ne sera appliquée sur l'objet ou ses descendants.

    •  
  • Associer : lien entre deux objets pour qu'ils soient considérés comme des équivalents et soient comparés.

    •  
  • Rompre l’association : défaire le lien entre deux objets pour qu'ils ne soient pas considérés comme des équivalents et ne soient pas comparés.

    •  
Certaines actions sont offertes pour les différences des propriétés d’objets :
     
  • Modifier : modification de la valeur des propriétés de l'objet source et cible.

    •  
  • Afficher la valeur : permet un affichage des valeurs sources et cibles.

    •  


Le rapport

Un rapport est toujours produit à la fin du processus d'intégration. Ce rapport présente les deux modèles qui ont été comparés, la date et l’heure, la liste de différences et les modifications appliquées (s’il y en a). Dans le but d’analyser l’impact d’éventuelles modifications, ce même rapport peut aussi être produit avant l'intégration.
