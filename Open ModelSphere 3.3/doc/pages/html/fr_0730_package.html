Le concept de paquetage


Un paquetage est un mécanisme tout usage pour organiser des éléments en groupe. Des paquetages peuvent être imbriqués dans d'autres paquetages. Un système peut être considéré comme un paquetage unique de haut niveau, contenant tous les autres objets du système à l’intérieur de ce paquetage.


Les propriétés du paquetage

Lors du choix d'un paquetage, cliquez ce dernier avec le bouton droit et, du menu contextuel, choisissez Propriétés. Dans la fenêtre de propriétés sous l’onglet conception, vous pouvez voir ceci :
  • Nom : nom physique du paquetage.

  • Alias : identifie un concept de façon différente (autre que par son nom).

  • Stéréotype UML : façade, framework, invariant, postcondition, precondition, stub, boundary, entity, control ou aucun.

  • Version : indiquez la version du paquetage. La version est affichée dans le cartouche du diagramme.

  • Auteur : indiquez le nom de l’auteur du paquetage. L’auteur est affiché dans le cartouche du diagramme.

  • Système cible : le système cible assigné est toujours Java.

  • Description : permet d’inscrire une description du paquetage sous forme de texte.
